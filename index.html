Meal Management System

Developed at: Instituto das Cidades Inteligentes (ICI)

The Meal Management System provides the management functionalities required by the Department of Education of Curitiba to control food distribution across hundreds of public schools in the city.

The project aims to support the Municipal Department of Education of Curitiba in managing and overseeing school meals for more than 450 educational units. The system provides features for educators, nutritionists, and other school staff to request and manage meals, including special diets for students with specific needs, such as food intolerances or allergies. It also enables the creation of monthly menus for each school, offering nutritional information and intelligent mechanisms that automatically substitute meals for students with dietary restrictions. Additionally, the Meal Management System supports the evaluation of meal quality, service performance, and includes financial tools to calculate payments to suppliers and generate detailed reports.

The application is built on distributed architecture principles. ASP.NET Core and Angular are the core technologies for the back-end and front-end, respectively. A background service, powered by Hangfire, is responsible for handling calculations and executing heavy processing tasks. The system uses PostgreSQL as its main relational database. This service communicates with the APIs and front-end applications through a RabbitMQ event bus and SignalR.

Throughout the development of this system, I worked on both the back-end and front-end. I contributed to building APIs, background processing, and database integration with PostgreSQL, as well as implementing the user interface and core features using Angular and Tailwind.

SITES - Transportation for Special Education

Developed at: Instituto das Cidades Inteligentes (ICI)

A portal that allows citizens of Curitiba to request transportation for students with special needs.

The solution includes features that provide education professionals with an overview of transportation requests through dashboards, ID card issuance, reporting tools, route visualization, and real-time vehicle tracking. Citizens can access the public portal to submit and monitor requests, while service providers can generate attendance lists, register attendance via QR Code, and manage staff and bus routes directly through the Company Portal.

The system was developed using .NET following the MVC pattern. It includes two portals: an administrative portal for managing transportation requests and generating reports, and a public-facing portal where citizens can request a vehicle.

The application uses PostgreSQL as its main database engine, and background services are handled through Hangfire for scheduled and asynchronous processing. The user interface is styled with Bootstrap and the front-end assets are bundled using Webpack.

To ensure quality and reliability, the system is covered with unit tests implemented using xUnit and NSubstitute.

Although my involvement in the project was brief, I was responsible for developing a significant portion of the portal, contributing directly to the implementation of its main features and overall structure.

CarreiraTI - IT Job Portal

Developed at: Code 21

CarreiraTI is a job portal designed to connect technology professionals with career opportunities in the IT sector. The platform allows both candidates and companies to register, providing features for job seekers to create profiles and apply for positions, while enabling organizations to publish and manage their open positions.

The project was initially developed using ASP.NET MVC, and later modernized through a complete front-end migration to Angular, improving scalability, performance, and user experience. I actively contributed to this migration process, helping to refactor legacy views, structure Angular components, and integrate them with the existing back-end.

CarreiraTI offers core features such as job postings, search and filtering capabilities, profile management, and company dashboards. The back-end is built with .NET API and manages business logic, authentication, and communication with a relational database. The front-end, developed with Angular, provides a responsive and intuitive interface for both job seekers and employers.

During the development, I worked across both the back-end and front-end, gaining experience with API development, component-based architecture in Angular, and implementing essential functionality for a job marketplace platform.

NopCommerce

Developed at: Code 21

The E-commerce Development Project was focused on building and customizing online stores using the open-source platform NopCommerce.
